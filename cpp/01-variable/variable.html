Type, variable, assignment, expression¶

Types¶

  • For each type, understand:

    • is a class type?

    • is numeric?

    • is ordered?

    • is signed?

    • range

    • related types

Expressions¶

  • operators

    • types

      • arithmetic

      • relational

      • logic

      • bit-wise

      • other: <<, >>, &, *, ., ->

    • = vs ==

    • relational && || vs bit-wise & |

    • bool vs int, asymmetric interconversion

      • true -> 1, false -> 0

      • non-zero -> true, 0 -> false

1// ==== operators ====
2
3// ---- wrong ----
4double a = 10;
5int b = a % 3;
6if (grade = 100)
7  cout << "perfect!";
8if (age < 10 & age > 1)   // this actually works well but is a wrong syntax
9  cout << "Young child\n";

  • type cast

    Will not change the value of the variable. Will change the type of the expression only.

    • assignment

    • mixed calculation

    • explicit type_cast<T>(var)

 1// ---- correct ----
 2int a = 10;
 3double b;
 4char c = 'L';
 5
 6b = a;
 7a = b;
 8b = a + c;
 9
10int total = 7;
11int count = 3;
12double average = type_cast<double>(total) / count;
13
14// convert to 3.210000
15double val = 3.213;
16double result = static_cast<int>(val * 100) / 100.0;
17
18// ---- wrong ----
19int total = 7;
20int count = 3;
21double average = total / count;  // loose precision, get 2.0, expect 2.3333...
22double average = static_cast<double>(total / count); // cast too late

  • Step-wise evaluation, in every step consider:

    • implicit/explicit type casting

    • precedence rule

    • pitfalls

      • % on double

      • lose precision in / integer divisions

 2
 3// ---- step-wise evaluation ----
 4when x = 2
 5  (x + 5) / 4
 6= (2 + 5) / 4
 7= 7 / 4
 8= 1
 9
10  static_cast<int>(19.5 / 3) + 5.0
11= static_cast<int>(6.5) + 5.0
12= 6 + 5.0
13= 11.0

15  ((17 / 2 + 19 % 3 - 3 * 2 + 4 * 3) >= (5 + 2 * 4)) && !((((4 * 5) + 1) /2) == (7 / 3 + 2 * 5 - 2) || !(4 >= 3))
