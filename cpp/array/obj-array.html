Object Array¶

Multiple arrays¶

  • store data in various arrays, same index means same record

    E.g. grades[i] and names[i] refers to the grade and name of a same student

    1const int NUM_STUDENT = 100; // max number of students
2double grades[NUM_STUDENT];
3string names[NUM_STUDENT];
4names[4] = "John Smith";
5grades[4] = 100;
6cout << "The 5th student in the list " << names[4] << " has a grade of " << grades[4];

  • old-fashion approach inherited from C language

Array of objects¶

  • Better alternative to the multiple array approach

 1struct Student {
 2  double grade;
 3  string name;
 4};
 5
 6Student students[NUM_STUDENT];
 7students[4].name = "John Smith";
 8students[4].grade = 100;
 9cout << "The 5th student in the list " << students[4].name << " has a grade of " << students[4].grade;
10
11class Student1 {
12  double grade;
13  string name;
14 public:
15  Student1(): name(""), grade(0) {}
16  Student1(string name, double grade): name(name), grade(grade) {}
17  string getName() { return name;}
18  double getGrade() { return grade;}
19};
20
21Student1 students1[NUM_STUDENT];  // default constructor called 100 times
22students1[4] = Student1("John Smith", 100);  // temporary object assignment
23cout << "The 5th student in the list " << students1[4].getName() << " has a grade of " << students1[4].getGrade();

Array of pointers to objects¶

  • Better efficiency of memory usage, less memory allocation

  • Complex syntax

Student1 *students2[NUM_STUDENT];  // 100 pointers
students2[4] = new Student1("John Smith", 100);
cout << "The 5th student in the list " << students2[4]->getName() << " has a grade of " << students2[4]->getGrade();
