Object ArrayÂ¶

When modeling a list of N records with M fields (a N * M 2D table), there are several data structures to choose from:

  1. Multiple arrays

  • store data in M arrays, each with size N, same index means same record

    E.g. grades[i] and names[i] refers to the grade and name of a same student

    1const int NUM_STUDENT = 100; // max number of students
2double grades[NUM_STUDENT];
3string names[NUM_STUDENT];
4
5names[4] = "John Smith";
6grades[4] = 100;
7cout << "The 5th student in the list " << names[4];
8cout << " has a grade of " << grades[4];

  • old-fashion approach inherited from C language

  1. Array of objects (instance of a class or a struct)

  • Better than the multiple array approach

  • Pre-allocate N objects

  • Each object has M fields

 1// struct example
 2struct Student {
 3  double grade;
 4  string name;
 5};
 6
 7Student students[NUM_STUDENT];
 8students[4].name = "John Smith";
 9students[4].grade = 100;
10cout << "The 5th student in the list " << students[4].name;
11cout << " has a grade of " << students[4].grade;
12
13// class example
14class Student1 {
15  double grade;
16  string name;
17public:
18  Student1(): name(""), grade(0) {}
19  Student1(string name, double grade): name(name), grade(grade) {}
20  string getName() { return name;}
21  double getGrade() { return grade;}
22};
23
24Student1 students1[NUM_STUDENT];  // default constructor called 100 times
25students1[4] = Student1("John Smith", 100);  // temporary object assignment
26cout << "The 5th student in the list " << students1[4].getName() << " has a grade of " << students1[4].getGrade();

  1. Array of pointers to objects

  • Pre-allocate N pointers

  • Efficient when the array is partially filled

  • Complex syntax

1Student1 *students2[NUM_STUDENT];  // 100 pointers
2for (int i = 0; i < NUM_STUDENT; i++)
3  students2[i] = nullptr;
4students[4] = new Student1("John Smith", 100);
5cout << "The 5th student in the list " << students2[4]->getName();
6cout << " has a grade of " << students2[4]->getGrade();
